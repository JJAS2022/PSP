Lectura de ficheros del Servidor Web

Lectura de Archivos con el Módulo fs

En Node.js, el módulo fs (sistema de archivos) es esencial para realizar operaciones de lectura y escritura en archivos del sistema. La función principal para leer archivos es readFile(), que toma como parámetros el nombre del archivo, la codificación (opcional) y una función de devolución de llamada que manejará los datos leídos. 


            const fs = require('fs');

            // Ejemplo de lectura de un archivo HTML con codificación 'utf8'
            fs.readFile('archivo.html', 'utf8', (err, data) => {
                if (err) {
                    // Si hay un error al leer el archivo, imprime el error en la consola
                    console.error(err);
                    return;
                }
                // Imprime el contenido del archivo en la consola
                console.log('Contenido del archivo:', data);
            });

En este ejemplo, el archivo 'archivo.html' se lee con la codificación 'utf8', que es común para archivos de texto. El tercer parámetro de readFile() es una función de devolución de llamada que se ejecutará una vez que la lectura del archivo esté completa. Esta función recibe dos parámetros: err para manejar posibles errores y data que contiene el contenido del archivo leído.

Es importante manejar los errores correctamente para garantizar que la aplicación no falle inesperadamente en caso de problemas durante la lectura del archivo. Por otro lado, al combinar la lectura de archivos con la capacidad de enviar estos datos como respuesta, podemos construir aplicaciones web dinámicas que sirven contenido específico desde el servidor hacia el cliente.
