File Reading in the Web Server

Reading Files with the fs Module

In Node.js, the fs (file system) module is essential for performing read and write operations on the system's files. The main function for reading files is readFile(), which takes parameters such as the file name, encoding (optional), and a callback function that will handle the read data. 


            const fs = require('fs');

            // Example of reading an HTML file with 'utf8' encoding
            fs.readFile('file.html', 'utf8', (err, data) => {
                if (err) {
                    // If there is an error reading the file, print the error to the console
                    console.error(err);
                    return;
                }
                // Print the content of the file to the console
                console.log('File content:', data);
            });

In this example, the 'file.html' file is read with the 'utf8' encoding, which is common for text files. The third parameter of readFile() is a callback function that will be executed once the file reading is complete. This function receives two parameters: err to handle potential errors and data containing the content of the read file.

Handling errors correctly is important to ensure that the application does not unexpectedly fail in case of issues during the file reading. On the other hand, by combining file reading with the ability to send this data as a response, we can build dynamic web applications that serve specific content from the server to the client.
